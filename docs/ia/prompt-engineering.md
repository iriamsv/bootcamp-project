# Prompt Engineering

En este documento voy a recopilar distintos prompts utilizados durante el desarrollo del proyecto.

Analizaré:
- Qué tipo de prompts funcionan mejor
- Cómo influyen en la calidad de las respuestas
- Diferentes técnicas como roles, restricciones o ejemplos

También incluiré ejemplos prácticos aplicados al código del proyecto.

## Prompt 1 – Rol (desarrollador senior)

"Actúa como un desarrollador senior y refactoriza este código"

Por qué funciona:
Definir un rol hace que la IA genere código más profesional y estructurado.


## Prompt 2 – Explicación paso a paso

"Explícame este código paso a paso como si fuera principiante"

Por qué funciona:
Obliga a la IA a simplificar la explicación y hacerla más clara.


## Prompt 3 – Refactorización

"Refactoriza esta función para que sea más limpia y legible"

Por qué funciona:
La IA se centra en mejorar estructura sin cambiar funcionalidad.


## Prompt 4 – Detección de errores

"Encuentra el error en este código y explícalo"

Por qué funciona:
La IA analiza el código de forma crítica y detecta fallos.


## Prompt 5 – Restricciones claras

"No cambies la funcionalidad, solo mejora la legibilidad del código"

Por qué funciona:
Evita que la IA rompa el comportamiento del programa.


## Prompt 6 – Generación de función

"Crea una función en JavaScript que filtre tareas completadas"

Por qué funciona:
Es específico y directo, lo que genera código útil rápidamente.


## Prompt 7 – Mejora de nombres

"Mejora los nombres de variables para que sean más descriptivos"

Por qué funciona:
La IA optimiza la claridad del código.


## Prompt 8 – Análisis de código

"Analiza este código y dime qué se puede mejorar"

Por qué funciona:
Permite obtener feedback general sobre calidad del código.


## Prompt 9 – Validación

"Añade validaciones a este formulario"

Por qué funciona:
La IA añade lógica adicional útil sin rehacer todo el código.


## Prompt 10 – Comparación

"Compara estas dos soluciones y dime cuál es mejor y por qué"

Por qué funciona:
Obliga a la IA a razonar y no solo generar código.

## Prompt extra – Bug real

"Mi calendario no funciona, revisa este código y dime el problema"

Por qué funciona:
La IA analiza el contexto real del problema y ayuda a detectar errores específicos del proyecto.
